Detailed Description

Dichiarazioni delle funzioni di test per i vari moduli del programma.

Questo file contiene le firme delle funzioni di test unitario, ciascuna dedicata alla verifica della correttezza e del comportamento atteso delle funzioni implementate nei rispettivi moduli: dati, utilitŕ, controlli, gestione file e funzionalitŕ principali.

◆ testControlli()

void testControlli ( )

Esegue i test unitari per il modulo controlli.

Questa funzione verifica il comportamento delle funzioni di validazione e controllo dei dati (es. formati di stringhe, ID duplicati, ecc.), definite in controlli.h e implementate in controlli.c.

Esegue i test unitari per il modulo controlli.

Questa funzione testa:

  • formatoNomeFile: verifica la correttezza del formato dei nomi dei file.
  • formatoStringa: controlla la presenza di spazi iniziali/finali o tabulazioni nelle stringhe.
  • stringaSoloNumeriODecimali: verifica se una stringa contiene solo numeri o decimali.
  • stringaSoloCaratteri: verifica se una stringa contiene solo caratteri alfabetici.
  • lunghezzaMassimaStringa: controlla la lunghezza massima consentita per una stringa.
  • controlloRigheFile: verifica che ogni riga di un file termini con un newline, inclusa l'ultima.
  • controlloNumeroDivisori: conta e valida il numero di separatori in una stringa di record.
  • validaRecordAnimale: funzione complessa che valida un'intera stringa di record di animale considerando tutti i criteri (formato, duplicati, lunghezze, tipi di dato).

Per i test sui file, vengono creati temporaneamente dei file (.txt) che vengono poi rimossi. Vengono utilizzate asserzioni (assert) per verificare la correttezza dei risultati in ogni test case.

◆ testDati()

void testDati ( )

Esegue i test unitari per il modulo dati.

Questa funzione verifica il corretto funzionamento delle strutture dati e delle operazioni ad esse associate (creazione, distruzione, accesso, modifica, ecc.) definite in dati.h e implementate in dati.c.

Esegue i test unitari per il modulo dati.

Questa funzione testa:

  • La creazione e l'inserimento di oggetti Animale in una Lista di animali.
  • La stampa della lista di animali.
  • La creazione e l'inserimento di oggetti Negozio in una lista globale di negozi.
  • La stampa della lista globale dei negozi.
  • La ricerca di un animale per ID all'interno di una lista, sia per casi di successo che di fallimento.

Vengono utilizzate asserzioni (assert) per verificare che le operazioni restituiscano i valori attesi e che le strutture dati siano manipolate correttamente. Alla fine, tutta la memoria allocata durante il test viene liberata.

◆ testFile()

void testFile ( )

Esegue i test unitari per il modulo file.

Questa funzione verifica la correttezza delle operazioni sui file, inclusa l'apertura, chiusura, lettura, scrittura e l'estrazione di token, definite in file.h e implementate in file.c.

Tests the apriFile function with an existing file in read mode.

This assertion verifies that apriFile returns 1 (success) when attempting to open a pre-existing file in "r" (read) mode.

Tests the chiudiFile function after a successful file open.

This assertion verifies that chiudiFile returns 1 (success) after a file has been successfully opened.

Tests the apriFile function with an existing file in write mode.

This assertion verifies that apriFile returns 1 (success) when attempting to open a pre-existing file in "w" (write) mode.

Tests the chiudiFile function after a successful file open in write mode.

This assertion verifies that chiudiFile returns 1 (success) after a file has been successfully opened in "w" mode.

Tests the apriFile function with a non-existent file in read mode.

This assertion verifies that apriFile returns 0 (failure) when attempting to open a non-existent file in "r" (read) mode.

Tests the chiudiFile function when no file is currently open.

This assertion verifies that chiudiFile returns 0 (failure) when attempting to close a file when no file is currently open.

Tests the letturaFile function.

This call attempts to read data from "negozio_1.txt" and populate a linked list of NodoNegozioPtr.

Tests the scritturaSingoloNegozioFile function.

This loop iterates through the loaded NodoNegozioPtr list and writes each node's data to "testScrittura.txt".

Tests the estrazioneToken function with a valid input string.

This assertion verifies that estrazioneToken correctly parses a well-formatted string and returns 1 (success). It also prints the extracted values for manual verification.

Tests the estrazioneToken function with an invalid input string.

This assertion verifies that estrazioneToken returns 0 (failure) when provided with a malformed string that cannot be parsed correctly.

◆ testFunzioni()

void testFunzioni ( )

Esegue i test unitari per il modulo funzioni.

Questa funzione verifica la logica di business principale dell'applicazione, inclusi i criteri di confronto, le funzionalitŕ di acquisizione, uguaglianza e ordinamento, e la gestione del menu, definite in funzioni.h e implementate in funzioni.c.

Tests the uguaglianzaNegozi function with two identical shops.

This call attempts to find common animals between two shops that have the exact same inventory. It then asserts that a non-NULL result (indicating common elements) is returned and prints the resulting common list.

Tests the uguaglianzaNegozi function with two shops having different inventories.

This call attempts to find common animals between two shops with entirely different inventories. It then asserts that a NULL result (indicating no common elements) is returned.

Tests the uguaglianzaNegozi function with shops containing internal duplicates.

This call attempts to find common animals between two shops where one or both lists of animals might contain duplicate entries (based on ID, name, etc.). It then prints the resulting common list or confirms no common elements are found.

Tests the funzionalitaUguaglianza function.

This function call processes the global list of shops and writes the results of equality comparisons to a file named "Uguaglianza.txt". Manual verification of the output file is required to confirm correctness.

Tests the funzionalitaOrdine function.

This function call processes the global list of shops and writes the results of the sorting operation to a file named "Ordine.txt". Manual verification of the output file is required to confirm correctness of the sorting logic.
